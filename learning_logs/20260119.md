## `<input>` 태그 사용하기
- 입력 받을 정보에 따라 여러 가지 type 속성이 있습니다. 제가 소개하는 것 이외에도 엄청 많아서 추후 mdn 공식 문서를 확인해보시는 것을 추천드립니다.

### 텍스트 관련 `<input>`
03_textinputtag.html 생성 및 사전 작업하겠습니다.

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body bgcolor="5f5f5f">
    <h1>textinputtag.html</h1>
    <h1>텍스트 관련 input 타입</h1>
    <form action="#">
        <label for="txtIp">text</label><br>
        <input 
            id="txtIp"
            type="text"
            placeholder="5 자 이하"
            maxlength="5"
        >
        <br><br>
        <label for="pwdIp">password</label><br>
        <input 
            id="pwdIp"
            type="password"
            placeholder="4자 이상"
            minlength="4"
        >
        <br><br>
        <label for="srcIp">search</label><br>
        <input 
            id="srcIp"
            type="search">
        <br><br>    <!-- type="tel"로 생성해보시오.-->
        <label for="tlIp">tel</label><br>
        <input 
            id="tlIp"
            type="text"
        >
        <br><br>
        <button type="submit">제출</button>
    </form>
</body>
</html>
```
이상의 html에서 type으로 1. text / 2. password / 3. search / 4. tel 을 사용했습니다. input 태그의 default는 text로 생성됩니다.(JS / python에서도 동일합니다)

1. text
    - maxlength / minlength를 확인할 수 있고, 이때 안내문을 출력해주는 것이 사용자 입장에서 좋겠네요. 그 때 사용할 수 있는 속성이 placeholder가 되겠습니다.
        - placeholder : 사용자가 안내 메시지 등을 미리 확인하고 입력할 수 있도록 도와주는 역할.
2. password
    - 우리가 무엇을 입력하는지 모르도록 글자가 점 형태로 가려집니다. 보안 설정이 필요할 때 사용.
    - button - type=submit이었을 때 4자 미만으로 입력했을 때 안내문/경고문이 출력되었습니다. 이는 브라우저마다 나타나는 방식이 다르기 때문에 js 등을 이용하여 추가 경고 팝업을 띄우는 방식을 주로 사용합니다.
3. search 
    - 다른 거 별로 차이 없어 보이는데 텍스트를 입력하면 입력 폼 오른쪽에 x 표시가 뜨고, 이거 클릭하면 입력 한 텍스트를 초기화해줍니다. 
    - 네이버 / 구글 등에 나오는 x 자 표시와 동일한 역할을 합니다. -> 근데 이것도 css로 처리하는 편.
4. tel
    - 전화번호 입력 폼입니다. 웹에서는 별만 차이 없고 모바일에서 실행하면 알아서 번호판 키패드가 팝업됩니다.

### 숫자 관련 `<input>`
04_numberinputtag.html 생성하고 초기 설정하시오.

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f">
  <h1>04_numberinputtag.html</h1>
  <h1>숫자 관련 인풋 타입</h1><!-- id=numIp / type = number / min=0 / max=10으로 만드시오.-->
  <label for="numIp">number</label><br>
  <input 
    id="numIp"
    type="number" 
    min="0"
    max="10"
    >
  <br><br><!-- id=rgIp / type = range / min=0 / max=100 / step=20으로 만드시오.-->
  <label for="rgIp">range</label><br>
  <input 
    id="rgIp"
    type="range"
    min="0"
    max="100"
    step="20"
  >
  <br><br><!-- id=dtIp / type = date / min="2020-01-01" / max="2030-12-31"으로 만드시오.-->
  <label for="dtIp">date</label><br>
  <input 
    id="dtIp" 
    type="date"
    min="2010-01-01"
    max="2030-12-31"
  >
  <br><br>
</body>
</html>
```
1. number - 입력 폼에 숫자만 넣을 수 있음. 화살표 생깁니다. 화살표 모양도 브라우저 따라 다릅니다.
2. range - 슬라이드 바를 왼쪽 오른쪽으로 조절하는 타입. step 속성에 숫자 단위를 입력하는데, JS와 함께 사용하는 편입니다.
3. date - 달력 모양이 나오는데 브라우저 마다 다르게 나오기 때문에 마찬가지로 react 등을 이용해서 사용하는 편입니다.

### 체크 관련 `<input>`

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <h2>checkbox</h2>
    <input 
      id="cbIp"
      type="checkbox"
      checked  
    >

    <h2>radio</h2>
    <input 
      type="radio"
      name="fruit"
      id="f_apple"
      value="apple"
      checked  
    >

    >
    <label for="f_orange">오렌지</label>
    <br>
    <input 
      type="radio"
      name="vege"
      id="v_carrot"
      value="carrot"
      checked
      >
    <label for="v_carrot">당근</label> <!-- 토마토 하나 / 가지(eggplant) 하나 만들겠습니다-->

      checked
      >

      type="radio"

  </form>
```

1. checkbox - 체크 박스 형태로 사용자 입력을 받음.
    - label 태그에서의 for 속성값 = input 태그에서의 id 속성값을 일 경우에 label 태그의 요소를 클릭했을 때 체크박스가 checked로 변경됨.
    - checked 속성 : 처음 페이지에 들어왔을 때 미리 체크되어있는 항목
2. radio - 라디오 버튼이라고 불리는 동그라미 형태의 버튼. 각 그룹에서 하나를 선택하면 다른 하나는 선택해제가 되는 것을 확인할 수 있습니다.
    - name 속성 : 그룹을 설정할 때 사용하는 속성으로, 동일 name 내에서 하나만 선택됩니다.
    - value 속성 : 결과를 서버로 넘기는 실제 데이터 값을 의미함. 실제 값은 프로그래밍에 적합한 영어나 코드 형식으로 보내기 때문에 label 태그 내의 요소가 한국어로 적힌 것과 달리 영어로 작성하는 편입니다.

### `<input>` 태그 공통 속성
07_commoninputtag.html 생성
```html
<!DOCTYPE html>
<html lang="en">

  <meta charset="UTF-8">

  <h1>인풋 요소(element) 공통(대부분) 속성</h1>

    >
    <br><br>

      disabled
    >
    <br><br>
    <label for="rqIp">required</label><br>
    <input 
      id="rqIp"
      type="text"
      placeholder="필수 입력"
      required
    >
    <br><br>
    <input 
      type="radio"
      name="fruit"
      id="f_apple"
      value="apple"
      checked
    >
    <label for="f_apple">사과</label>
    <input 
      type="radio"
      name="fruit"
      id="f_grape"
      value="grape"
    >
    <label for="f_grape">포도</label>
    <input 
      type="radio"
      name="fruit"
      id="f_orange"
      value="orange"
      disabled
    >
    <label for="f_orange">오렌지(품절)</label> <br><br><br>
    <button type="submit">제출</button>
  </form>
</body>
</html>
```

1. value 속성 : input 내에 자체적으로 텍스트가 입력되어있음.
    - 웹 사이트에서 이전에 입력한 내 아이디가 미리 떠 있는 것과 유사합니다.
    - 안내문만 표현하는 placeholder와 달리 실제로 제출 버튼을 누르게 되면 value 속성에 들어간 속성값이 그대로 서버에 전송됩니다.
2. autofoucs 속성 : 어떤 페이지가 떴을 때(렌더링됐을 때) 클릭하지 않아도 자동으로 특정 요소가 선택되도록 지정(로그인 페이지 시 id input창에 커서 깜빡이는게 예시).
3. readonly 속성 : 읽기만 가능하고 입력 값은 결과로 서버에 전송됨.
4. disabled 속성 : 사용자가 값을 입력할 수도 없고 결과가 전송되지도 않음.
5. required 속성 : 반드시 결과값을 입력해야하는 것을 의미함.

## 기타 다른 태그 사용하기
- 수업하지 않았지만 제한적으로 사용될 수 있는 태그들을 일부 소개합니다. 하지만 기본적으로 mdn의 문서를 한 번 확인하는 것을 추천합니다.
### `<textarea>`
    - input 태그와 달리 긴 분량의 텍스트, 즉 텍스트 입력 영역이 페이지에서 여러 줄 이상을 차지하도록 할 때 사용하는 태그.
    - cols속성(너비) / rows속성(줄 수)를 통해 텍스트 영역의 크기를 설정함.
    - 08_textareatag.html 생성하시고, 초기 설정하시고,
    - cols="64" / rows="5"로 한 번 만들어보세요.

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f">
  <h1>08_textareatag.html</h1>
  <h1>textarea 태그</h1>
  <label for="message">메시지를 입력하세요</label> <br>
  <textarea id="message" cols="64" rows="5">여기에 기본값을 입력할 수 있습니다.</textarea>
</body>
</html>
```
- 주의 사항
    - textarea 태그의 경우 input 태그의 여러줄 버전이라고 생각하기 쉽고, 그게 사실 맞습니다. 하지만 value=""를 통해 default 값을 입력할 수 있는 input 태그와 달리 여는 태그 / 닫는 태그로 구성되어있기 때문에 내부 요소로 default 값을 입력할 수 있습니다.
### `<option>`
09_optiontag.html 생성 후 기본 설정하시오.
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f">
  <h1>09_optiontag.html</h1>
  <h1>옵션들을 사용하는 태그</h1>
  <h2>select, option 태그</h2>
  <label for="lang">language</label> <br>
  <select id="lang">
    <option value="">-- 언어 선택 --</option>
    <option value="html">HTML</option>
    <option value="css">CSS</option>
    <option value="js">JavaScript</option>
    <option value="ts">TypeScript</option>
  </select>
  <br><br>
  <h2>optgroup 태그</h2>
  <label for="shopping">쇼핑 목록</label> <br>
  <select id="shopping">
    <optgroup label="과일">
      <option value="f_apple">사과</option>
      <option value="f_grape">포도</option>
      <option value="f_orange">오렌지</option>
    </optgroup>
    <optgroup label="채소">
      <option value="v_carrot">당근</option>
      <option value="v_tomato">토마토</option>
      <option value="v_eggplant">가지</option>
    </optgroup>
    <optgroup label="육류">
      <option value="m_beef">소고기</option>
      <option value="m_pork">돼지고기</option>
      <option value="m_chicken">닭고기</option>
    </optgroup>
  </select>
  <br><br>
  <h2>datalist 태그</h2>
  <label for="job">현재 직업</label><br>
  <input id="job" list="jobs">
  <datalist id="jobs">
    <option value="학생"></option>
    <option value="디자이너"></option>
    <option value="퍼블리셔"></option>
    <option value="개발자"></option>
  </datalist>
</body>
</html>
```
1. select + option 태그 : 사용자 눈에 보이는 결과와는 별개로 value 속성을 이용하여 전송될 결과값을 추가해줘야합니다. 그래서 아무 것도 선택하지 않는 상태인 '-- 언어 선택 --'에는 value=""로 처리했습니다.
    - default 값으로 무언가 하나를 보여주고 싶다면 option 태그 내에 selected 속성을 지정해줄 수도 있습니다.
    - select 내에 multiple을 지정할 수도 있습니다.
    - shift 키 / ctrl 키를 통해서 다중 선택이 가능하긴 하지만 별도의 설명이 없다면 사용자가 알아채기 힘들기 때문에(시인성이 낮기 때문에) 해당 방식은 실무에서는 거의 사용되지 않고, 차라리 그럴거면 input 태그의 checkbox 속성을 이용해서 그냥 여러 개 선택하는 게 낫습니다.
2. select + optgroup 태그 :
    - 옵션 안에 있는 내용들을 그룹화하는 기능으로, optgroup에 있는 내용은 선택이 불가능합니다.
3. datalist 태그 :
    - label + input 태그로 동일하게 보이지만 그 밑에 datalist 태그가 있습니다. datalist 내의 option 태그들은 추천 답안을 미리 넣어둔 것이라고 할 수 있습니다. 이 때 중요한 점은 label의 for 속성과 input의 id 속성을 일치시켰고, input의 list 속성과 datalist의 id 속성을 일치시켰다는 점입니다.

```html

  >
  </iframe>

</html>
```
- width / height 를 지정할 수 있는데 -> 절대값 넣었으니까 모바일에서도 이 사이즈라는 점이 문제
- 특별한 경우가 아니면 권장하지 않습니다.
* 문제점
1. 보안 문제 - 누군가가 자신의 웹 페이지 안에 나의 웹 사이트를 full 화면으로 넣게 된다면 사실상 사이트를 복사한 것과 동일한 효과를 가지게 되고 id / password등을 탈취할 수 있습니다. 이를 clickjacking 이라고 합니다. 그래서 다른 웹 사이트에서 자신의 웹 사이트를 열지 못하도록 조치할 필요가 있습니다.

2. 사용성 저하 - 해당 사이트에 직접적으로 들어간게 아니라 다른 사이트를 통해서 들어가는 것이기 때문에 뒤로 가기 등이 적용되지 않습니다.

3. SEO에 최적화되지 않음. - 사이트 내용이 직접 들어간게 아니라 src 속성 내에 사이트 주소가 들어가있기 때문에 SEO와 관계없습니다.

4. 브라우저 새창 문제 - 한 화면에 여러 사이트를 불러오는 것과 같기 때문에 RAM 많이 잡아먹습니다.
